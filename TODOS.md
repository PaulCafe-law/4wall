# Product Roadmap Backlog

## Phase 1: Operable Product Skeleton

- Make `Overview` a real daily landing page for customers and internal users
- Add explicit artifact publication state, timestamps, and failure reasons to mission delivery views
- Strengthen customer team / invite workflows
- Upgrade `Support` and `Live Ops` from beta lists to usable internal workspaces
- Consolidate staging / production release acceptance into one checklist

## Phase 2: SaaS Core

- Add signup, email verification, password reset, and org create/join flows
- Add hosted billing and subscription workflows
- Add notification preferences and in-app/email notification delivery
- Harden abuse controls, idempotency, and self-serve audit coverage

## Phase 3: Internal Ops Console

- Formalize flight session list, telemetry freshness, video metadata, and control timeline
- Expand support queue with mission, artifact, telemetry, bridge, and billing exceptions
- Add internal-only mission-level control intents and acknowledgements
- Add remote planning workflows for internal users

## Phase 4: Trust, Delivery, and Scale

- Add delivery summary, artifact history, and customer-facing failure explanations
- Add site / mission / billing / incident history and exports
- Add deploy canary, health dashboard, alert routing, and rollback evidence
- Finalize role matrix, support-access policy, and audit retention/export
- Add onboarding walkthroughs, help center, and final language cleanup
